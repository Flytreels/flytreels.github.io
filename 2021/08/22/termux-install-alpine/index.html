在Termux上安装纯的AlpineLinux

Flyteee   in Earth
Sun Aug 22 2021
  • Linux
  • Termux
  • Android

这种方法也同样适用于其他的发行版，比如Arch Debian等等

# 前言

这里可能就会问道了，Termux不是有很多的安装Linux发行版的工具吗（proot-distro,anlinux等等），为什么还需要这样麻烦安装？
当然肯定有优势

  • [√] 从官方获取最小体积Rootfs
  • [√] 安全，快捷，更新迅速，方便后续扩展
  • [√] 不被第三方魔改或者加入风险

# 下载Rootfs

一般下载Rootfs都是从官方软件源或者镜像站或者LXC镜像站下载

这里下载alpine的rootfs只需要在官方镜像站就能下载到，有些发行版只能从lxc-images下载到（当然你也可使用各种Bootstrap工具去构建一个最新的）

Alpine 3.14下载 (opens new window)
一般情况顺藤摸瓜，看一下releases Rootfs images这类的文件夹

rootfs即根文件系统，一般情况以tar.gz或tar.xz格式打包，下载后需要解压到目标目录，大多情况为容器所使用，而我们在Termux上的chroot或者proot就都属于容器类型了，不能算是完整的系统，也没有真正模拟出一个计算机去运行发行版

# 安装ROOTFS

使用wget命令下载rootfs
目录随意，别是内部存储或者SD卡就行

wget https://mirrors.bfsu.edu.cn/alpine/v3.14/releases/aarch64/alpine-minirootfs-3.14.0_rc4-aarch64.tar.gz # 下载rootfs到当前目录
mkdir alpine # 新建rootfs存放文件夹（名称随意）
proot --link2symlink tar xzvf alpine-minirootfs-3.14.0_rc4-aarch64.tar.gz -C alpine # 解压rootfs，同时使用proot的link2symlink功能模拟硬链接（没ROOT使用此方法）

执行之后就安装到alpine文件夹辣！

# 启动容器

同时启动容器也非常简单，你也可以把他写进一个sh文件或者.bashrc让他在termux自启

unset LD_PRELOAD
proot -b /sys -b /proc -b /dev -r ./alpine/ --sysvipc --link2symlink -0 -w /root /bin/ash -l

执行上述命令就能启动容器内的shell

参数 解释
-b 挂载目录
-r 指定Rootfs目录
--sysvipc 在 proot 中处理 System V IPC 系统调用
--link2symlink 用符号链接替换硬链接，假装它们真的是硬链接
-0 模拟ROOT权限
-w 设置初始目录
/bin/ash -l 将ash作为进入容器环境后执行的第一个程序

挂载 /proc /dev 是必须的，否则容器内的大多数程序都无法正常工作！

# 基本配置

一般默认的rootfs都是最基本的没有任何配置过的，一般需要配置DNS和软件源即可 使用以下命令即可配置DNS为 8.8.8.8的谷歌DNS

echo "nameserver 8.8.8.8">/etc/resolv.conf
